# 実機検証チェックリスト — Video to Live Photo

最終更新: 2026-04-12

このチェックリストは、Windows 10 から EAS Build 経由で生成した Development Build / Production Build を **iPhone 実機**に入れて Live Photo 作成フローを検証するときに使います。シミュレータでは Live Photo の最終判定が当てにならないため、**実機確認が唯一の受け入れ条件**です。

> **Mac 不要前提**: このチェックリストは Xcode・Console.app・`xcrun` を**一切使いません**。ネイティブ側の診断はすべてアプリ内 Debug 画面（`Settings → デバッグ情報を開く`）と Metro / EAS Build ログで完結します。

## 確定値

| 種類 | 値 |
| --- | --- |
| App Display Name | `Video to Live Photo` |
| iOS Bundle Identifier | `com.gen.videotolivephoto` |
| 非消耗型 IAP Product ID | `com.gen.videotolivephoto.premium.hq_unlock` |

## 0. 前提条件

- [ ] `git status` がクリーン（もしくはネイティブ変更済みで commit + push 済み）
- [ ] `npm run lint` / `typecheck` / `test` / `check:marketing` / `check:iap` が全通過
- [ ] `scripts\windows\check-placeholders.ps1` が Green
- [ ] EAS にログイン済み（`npx eas whoami`）
- [ ] Apple Developer Program に加入済み、Bundle ID `com.gen.videotolivephoto` が登録済み
- [ ] Sandbox テスト用 Apple ID を作成済み（課金フロー検証用）
- [ ] iPhone 実機 (iOS 17 以上を推奨)

## 1. Development Build 作成

- [ ] ネイティブ変更がある場合は `--clear-cache` を付ける
  ```powershell
  npx eas build --profile development --platform ios --clear-cache
  ```
- [ ] ビルド成功の通知を受け取る（EAS の web dashboard も確認）
- [ ] iPhone で QR コードを開いて「インストール」
- [ ] 「設定 → 一般 → VPN とデバイス管理」で開発者プロファイルを信頼

## 2. Metro サーバ起動

- [ ] `npx expo start --dev-client` を起動
- [ ] Windows ファイアウォールで `node.exe` の LAN 通信を許可
- [ ] iPhone の Development Build アプリから Metro サーバへ接続
- [ ] ログに `Running "main" with ...` が出ることを確認

## 3. Live Photo 作成フロー

### 3.1 パーミッション
- [ ] 初回起動時、写真ライブラリ `.addOnly` 権限ダイアログが出る
- [ ] 許可後、ダイアログが再表示されない
- [ ] 拒否した場合、エラー文言「写真ライブラリへのアクセスが必要です」が表示される

### 3.2 動画選択
- [ ] 「動画を選ぶ」ボタンから写真ライブラリピッカーが起動する
- [ ] 縦動画・横動画のいずれも取り込める
- [ ] HEVC (H.265) / H.264 双方の動画が取り込める

### 3.3 トリム
- [ ] スライダーで 0.5〜3 秒の範囲を指定できる
- [ ] トリム確定後に静止画（キーフレーム）が生成される

### 3.4 書き出し
- [ ] 「標準画質で続ける」から保存ができる
- [ ] 「広告を見て高画質を 1 回試す」経由で 1 回だけ高画質保存ができる（Sandbox/テスト広告で確認）
- [ ] 「高画質を買い切りで解放」から IAP 購入 → 以降は広告なしで HQ 保存できる
- [ ] 購入復元（Restore Purchase）が動作する

### 3.5 アプリ内 Debug 画面で成功を確認
Xcode の代替として、`Settings → デバッグ情報を開く`（`__DEV__` ビルドのみ表示）で以下を確認します。

- [ ] 保存直後に `Last Save (success)` セクションに値が入る
  - [ ] `localIdentifier` に非空の文字列（`xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx/L0/001` 形式）
  - [ ] `contentIdentifier` に UUID が入っている
  - [ ] `at` が直近のタイムスタンプ
- [ ] `Log Buffer` セクションに `NativeLivePhotoBridge.saveLivePhoto <- { localIdentifier: ..., contentIdentifier: ... }` が流れる
- [ ] `Copyable Blob` を長押し → 選択 → コピーで、全情報を一括で送信・保存できる

### 3.6 Metro ログでネイティブ側工程を確認
Windows ターミナルの `npx expo start --dev-client` 画面に以下が順番で流れます（全てネイティブが `print(...)` で吐いた行を Metro が中継するもの）。

- [ ] `[ExpoLivePhotoExporter] performSave start`
- [ ] `[ExpoLivePhotoExporter] assigned contentIdentifier=<UUID>`
- [ ] `[ExpoLivePhotoExporter] tagged still written -> .../tagged.jpg`
- [ ] `[ExpoLivePhotoExporter] tagged movie written -> .../tagged.mov`
- [ ] `[ExpoLivePhotoExporter] asset created localIdentifier=...`
- [ ] Debug 画面の `contentIdentifier` と Metro ログの `contentIdentifier` が**同一 UUID**
- [ ] **エラー時は `ERR_...` コードがどの工程で出たかを記録**（Debug 画面 `Last Error` + Metro 双方）

## 4. Photos アプリでの Live Photo 判定

- [ ] Photos アプリを開いて最新 1 件を選択
- [ ] サムネイルまたは詳細画面の右上に **LIVE バッジ**がある
- [ ] **画面を長押し**すると動画部分が再生される
- [ ] 再生を離すとキーフレームに戻る
- [ ] アルバム「Live Photos」配下に入っている

## 5. 壁紙設定フロー（ユーザー手順の手動確認）

- [ ] 保存完了画面に「設定 → 壁紙 → 新しい壁紙を追加 → 写真 → Live Photo」の 4 ステップ案内が出る
- [ ] アプリ内文言に禁止表現（「自動設定」「ワンタップで壁紙」等）が**一切含まれない**
- [ ] 設定 App から Live Photo を壁紙にできる（手動で実行して確認）

## 6. 課金フロー (Sandbox)

- [ ] 設定 App で Sandbox Apple ID を設定
- [ ] アプリで買い切り購入 → 成功ダイアログ
- [ ] アプリ再起動後も HQ ロックが解除されたまま
- [ ] アプリ再インストール後、「購入を復元」で HQ ロックが再解除される
- [ ] 広告表示が完全に消えている

## 7. 広告フロー

- [ ] 買い切り未購入時のみ「広告視聴」ボタンが出る
- [ ] ボタンを**明示的にタップ**しないと広告が出ない（進捗画面を遮らない）
- [ ] 広告読み込み失敗時は標準画質導線へフォールバック
- [ ] 広告視聴完了後、次の 1 回だけ HQ 保存ができる
- [ ] HQ 保存後、trial フラグが消費される（2 回目はまた広告が必要）

## 8. 切り分けチェック（失敗時） — Windows 完結

Live にならない場合、以下の順で潰す。**全て Windows から確認可能**です。

1. [ ] `Settings → デバッグ情報を開く` で `Last Error` を確認
   - [ ] `code` がどの `ERR_...` か記録
   - [ ] `message` の内容を記録（必要なら `Copyable Blob` を長押しコピー）
2. [ ] Metro コンソール（`npx expo start --dev-client` 画面）に対応する `[ExpoLivePhotoExporter] ...` 行が出ているか確認
3. [ ] Debug 画面に `Last Save (success)` が一度でも入ったか確認
   - [ ] `localIdentifier` と `contentIdentifier` の両方が返っているか
4. [ ] Development Build が最新 SHA を反映しているか
   - PowerShell で `git rev-parse HEAD` を実行
   - `npx eas build:list --platform=ios --status=finished --limit=3` でビルド先頭の `gitCommitHash` と一致するか確認
   - 一致しない場合: ネイティブ変更時は再ビルド（`npx eas build --profile development --platform ios --clear-cache`）
5. [ ] EAS Build 自体が失敗していないか
   - `npx eas build:view <build-id>` で Xcode Archive ログを読む（ブラウザ不要）
   - または `start https://expo.dev/accounts/<account>/projects/video-to-livephoto/builds/<build-id>` でログ画面を開く
6. [ ] Photos アプリで **LIVE バッジ**が出ているか
   - バッジ無し → 条件 (A) JPEG tag か (D) PHAssetCreationRequest が欠落 → Metro の `tagged still written` 行と `asset created` 行を再確認
   - バッジ有り / 動かない → 条件 (B) MOV content identifier か (C) still-image-time が欠落 → Metro の `tagged movie written` 行を再確認
7. [ ] 入力動画が極端に短くないか（0.5 秒未満だと still-image-time が成立しない）
8. [ ] iCloud 写真同期の遅延ではないか（iPhone を Wi-Fi から切断し、アプリ内で新規作成してから再確認）
9. [ ] HEIC 入力で MakerApple dict が剥がれていないか（疑わしければ JPEG に変換して再実行）

> **Xcode は不要です**。上記ステップ 1〜5 がすべて Windows 内で完結します。iPhone を Mac に繋ぐ代わりに、Metro ログ＋アプリ内 Debug 画面＋`eas build:view` の 3 点セットで同じ情報が揃います。

## エラーコード早見表

| Code | 意味 | 対応 |
| --- | --- | --- |
| `ERR_PHOTO_PERMISSION_DENIED` | 写真ライブラリへの `.addOnly` 権限が無い | 設定 App で許可し直し |
| `ERR_INVALID_SOURCE_URI` | `movUri` / `stillUri` が空または無効 | JS 側の入力を確認 |
| `ERR_STILL_LOAD_FAILED` | `CGImageSourceCreateWithURL` が失敗 | 入力 JPEG/HEIC ファイルの破損 |
| `ERR_STILL_WRITE_FAILED` | `CGImageDestinationCreateWithURL` が失敗 | 一時ディレクトリ書き込み失敗 |
| `ERR_STILL_FINALIZE_FAILED` | `CGImageDestinationFinalize` が false | 稀。再実行で解消することあり |
| `ERR_MOVIE_TRACK_LOAD_FAILED` | `loadTracks(.video)` が例外 | 入力 MOV の破損 |
| `ERR_MOVIE_VIDEO_TRACK_MISSING` | ビデオトラック 0 本 | 入力素材が音声のみ |
| `ERR_MOVIE_READER_CREATE_FAILED` | `AVAssetReader` init 失敗 | 入力形式が想定外 |
| `ERR_MOVIE_WRITER_CREATE_FAILED` | `AVAssetWriter` init / canAdd 失敗 | 出力先が書き込めない |
| `ERR_MOVIE_START_WRITING_FAILED` | `writer.startWriting()` が false | metadata spec の不整合 |
| `ERR_MOVIE_VIDEO_APPEND_FAILED` | `videoInput.append()` が false | サンプルバッファ不正 |
| `ERR_MOVIE_FINISH_WRITING_FAILED` | `finishWriting` 後 status が completed 以外 | passthrough 失敗 / metadata 追記失敗 |
| `ERR_ASSET_CREATION_FAILED` | `PHAssetCreationRequest` 後 `localIdentifier` が空 | 権限または資産追加順序の問題 |
| `ERR_LIVE_PHOTO_EXPORT_FAILED` | 上記以外の未知のエラー | `message` を確認して切り分け |

すべて `codeToAppError` (`src/services/NativeLivePhotoBridge.ts`) で `AppError.exportFailed` へ集約され、UI には「書き出しに失敗しました。もう一度お試しください。」と表示されます。`underlying` には raw code が残るので、ログからは工程の特定が可能です。
